# LFX V2 Mailing List Service

The LFX v2 Mailing List Service is a lightweight proxy that exposes GroupsIO mailing list operations to the rest of the LFX platform. It forwards requests to the ITX API (`https://api.prod.itx.linuxfoundation.org`) and translates between LFX v2 project/committee UUIDs and the v1 SFIDs required by ITX.

## Quick Start

### For Deployment (Helm)

Both flows below require a Kubernetes secret with ITX credentials. If the `lfx` namespace doesn't exist yet:

```bash
kubectl create namespace lfx
```

Create the secret (values are in 1Password → **LFX V2** vault):

```bash
kubectl create secret generic lfx-v2-mailing-list-service -n lfx \
  --from-literal=ITX_CLIENT_ID="<value-from-1password>" \
  --from-literal=ITX_CLIENT_PRIVATE_KEY="<value-from-1password>" \
  --from-literal=ITX_AUTH0_DOMAIN="<value-from-1password>" \
  --from-literal=ITX_AUDIENCE="<value-from-1password>"
```

#### Deploy from GHCR (no local code changes)

```bash
make helm-install
```

#### Deploy a local build (with code changes)

```bash
cp charts/lfx-v2-mailing-list-service/values.local.example.yaml \
   charts/lfx-v2-mailing-list-service/values.local.yaml

make docker-build
make helm-install-local
```

### For Local Development

1. **Prerequisites**
   - Go 1.24+
   - Make

2. **Clone and Setup**

   ```bash
   git clone https://github.com/linuxfoundation/lfx-v2-mailing-list-service.git
   cd lfx-v2-mailing-list-service
   make setup
   ```

3. **Configure Environment**

   ```bash
   export AUTH_SOURCE=mock
   export JWT_AUTH_DISABLED_MOCK_LOCAL_PRINCIPAL="test-admin"

   # ITX proxy (required in non-mock mode)
   export ITX_BASE_URL="https://api.prod.itx.linuxfoundation.org"
   export ITX_CLIENT_ID="<your-client-id>"
   export ITX_CLIENT_PRIVATE_KEY="<your-rsa-private-key-pem>"
   export ITX_AUTH0_DOMAIN="<your-auth0-domain>"
   export ITX_AUDIENCE="<your-audience>"

   # ID mapping via NATS (optional — omit to use pass-through no-op mapper)
   export NATS_URL="nats://localhost:4222"
   ```

4. **Run the Service**

   ```bash
   make run
   ```

## Architecture

The service is a thin proxy between LFX platform consumers and the ITX GroupsIO API:

```
LFX Client  →  mailing-list-service  →  ITX API (groupsio_*)
                     ↕ (v1/v2 ID mapping via NATS)
```

- **API Layer**: Goa-generated HTTP handlers and OpenAPI specs
- **Service Layer** (`internal/service/itx/`): Maps v2 UUIDs → v1 SFIDs on requests, and v1 SFIDs → v2 UUIDs on responses
- **Domain Layer** (`internal/domain/`): `IDMapper` and `ITXGroupsioClient` interfaces, domain error types, ITX models
- **Infrastructure Layer**:
  - `internal/infrastructure/proxy/` — Auth0 M2M OAuth2 HTTP client (RS256 private-key assertion, token caching)
  - `internal/infrastructure/idmapper/` — NATS-based v1/v2 ID mapper with no-op fallback
  - `internal/infrastructure/auth/` — JWT authentication (Heimdall)

### ID Mapping

The API accepts LFX v2 UUIDs for projects and committees. Before forwarding requests to ITX, the service maps these to v1 SFIDs using NATS request/reply on the `lfx.lookup_v1_mapping` subject. Responses from ITX carry v1 SFIDs which are mapped back to v2 UUIDs before being returned to the caller.

If `NATS_URL` is not set (or `ID_MAPPING_DISABLED=true`), a no-op mapper is used that passes IDs through unchanged — useful for local development when the v1-sync-helper service is unavailable.

### Authentication to ITX

The proxy authenticates to ITX using Auth0 machine-to-machine (M2M) OAuth2 with a private-key JWT assertion (RS256). Tokens are automatically cached and refreshed via `oauth2.ReuseTokenSource`.

## Development

### Make Targets

| Target | Description |
|--------|-------------|
| `make all` | Full pipeline: setup → lint → test → build |
| `make setup` | Install dependencies and Goa CLI |
| `make apigen` | Regenerate code from design files |
| `make build` | Build the binary |
| `make run` | Build and run locally on port 8080 |
| `make test` | Run all tests with race detection and coverage |
| `make lint` | Run golangci-lint |
| `make docker-build` | Build Docker image |
| `make docker-run` | Run Docker container locally |
| `make helm-install` | Deploy to Kubernetes (GHCR image) |
| `make helm-install-local` | Deploy to Kubernetes (local image, mock auth) |

### Modifying the API

1. Edit `cmd/mailing-list-api/design/`
2. Run `make apigen` to regenerate `gen/`
3. Update the implementation in `cmd/mailing-list-api/service/mailing_list_api.go`
4. Run `make test` to verify

### Testing

```bash
make test

# Specific package
go test -v ./internal/service/itx/...

# With coverage report
go test -v -race -coverprofile=coverage.out ./...
go tool cover -html=coverage.out
```

Tests use hand-rolled mock structs with function fields — no external mock libraries required. See `internal/service/itx/mocks_test.go` for the pattern.

## Configuration

### Core

| Variable | Description | Default |
|----------|-------------|---------|
| `PORT` | HTTP server port | `8080` |
| `LOG_LEVEL` | Log level (`debug`, `info`, `warn`, `error`) | `info` |

### Authentication (inbound)

| Variable | Description | Default |
|----------|-------------|---------|
| `AUTH_SOURCE` | `jwt` or `mock` | `jwt` |
| `JWKS_URL` | JWKS endpoint for JWT verification | Heimdall cluster URL |
| `JWT_AUDIENCE` | JWT audience claim | `lfx-v2-mailing-list-service` |
| `JWT_AUTH_DISABLED_MOCK_LOCAL_PRINCIPAL` | Principal returned by mock auth | `""` |

### ITX Proxy (outbound)

| Variable | Description | Required |
|----------|-------------|----------|
| `ITX_BASE_URL` | ITX API base URL | Yes |
| `ITX_CLIENT_ID` | Auth0 client ID for M2M auth | Yes |
| `ITX_CLIENT_PRIVATE_KEY` | RSA private key in PEM format for client assertion | Yes |
| `ITX_AUTH0_DOMAIN` | Auth0 domain (e.g. `example.auth0.com`) | Yes |
| `ITX_AUDIENCE` | OAuth2 audience for token requests | Yes |

### ID Mapping

| Variable | Description | Default |
|----------|-------------|---------|
| `NATS_URL` | NATS server for v1/v2 ID lookup | `""` (no-op mapper) |
| `ID_MAPPING_DISABLED` | Set to `true` to force no-op mapper | `false` |

When `NATS_URL` is empty or `ID_MAPPING_DISABLED=true`, IDs are passed through unchanged. This is safe for local development but means ITX receives v2 UUIDs which it will not recognise in production.

### Local Development Example

```bash
# Run in fully mocked mode (no ITX calls, no NATS required)
export AUTH_SOURCE=mock
export JWT_AUTH_DISABLED_MOCK_LOCAL_PRINCIPAL="test-admin"
export ID_MAPPING_DISABLED=true
make run
```

## Deployment

### Kubernetes Secret

```bash
kubectl create secret generic lfx-v2-mailing-list-service -n lfx \
  --from-literal=ITX_CLIENT_ID="<value>" \
  --from-literal=ITX_CLIENT_PRIVATE_KEY="<value>" \
  --from-literal=ITX_AUTH0_DOMAIN="<value>" \
  --from-literal=ITX_AUDIENCE="<value>" \
  --dry-run=client -o yaml | kubectl apply -f -
```

### Helm Values Files

- `charts/lfx-v2-mailing-list-service/values.yaml` — Default config; pulls image from GHCR. Used by `make helm-install`. Do not commit local overrides here.
- `charts/lfx-v2-mailing-list-service/values.local.yaml` — Not tracked by git. Copy from the example file for local dev.
- `charts/lfx-v2-mailing-list-service/values.local.example.yaml` — Example override for running a locally-built image.

**⚠️ Security Warning**: Never use `AUTH_SOURCE=mock` in production.
